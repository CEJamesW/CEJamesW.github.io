Project Scope

项目简介

本系统围绕“高考志愿填报管理系统”场景设计，实现学生信息、班级信息、兴趣课程、大学、院系、专业、学生志愿和录取结果的统一管理。

程序仅使用控制台进行交互，使用 Maven 管理依赖，SQLite 进行数据持久化，分层架构组织代码。

MAINaidai.muyun.gaokao.Main
RUNrun.bat
ARCHui / service / dao / entity

System Goals

建设目标

10每名学生最多填报大学
3每所大学最多选择专业
5每门兴趣匹配课程加分
1一次录取生成结果

系统设计围绕题目要求展开：覆盖学生信息、大学院系专业、平行志愿填报、录取分配和结果查询，确保每个要求都能在控制台中完成演示。

Functional Modules

核心功能

学生基本信息管理新增、修改、删除、查询学生，维护学号、姓名、性别、成绩和班级。
学校院系专业管理管理大学、院系、专业、容量和专业关联兴趣课程。
志愿平行志愿填报限制大学数量、专业数量，并记录是否服从专业调剂。
录取自动分配与查询按规则生成录取结果，支持学校和学生双向查询。
演示初始化与重置首次运行自动创建数据库，答辩时可一键恢复演示数据。

Admission Algorithm

录取算法核心思想

综合录取分 高考成绩 + 兴趣课程匹配数 × 5
排序综合分降序，同分按学号升序。
志愿按学生填报顺序尝试大学。
容量学校与专业均需要有剩余名额。
调剂所选专业满额时，服从调剂进入同校其他专业。

录取算法模拟真实场景，兼顾 高考成绩 与 兴趣课程。
